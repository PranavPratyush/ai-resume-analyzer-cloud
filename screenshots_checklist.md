# Screenshots Checklist for Assignment Report

To complete your assignment report, you need to take the following screenshots from your browser and GitHub account:

### 1. Application Screenshots
- [ ] **Home Page:** Your app running on Hugging Face (show the URL `huggingface.co/spaces/00pranav00/resume-analyze`).
- [ ] **Analysis Result:** Upload a resume and show the skills/recommendations being generated.
- [ ] **Admin Dashboard:** Log in as admin and show the data table and analytical charts.

### 2. Deployment & CI/CD Screenshots
- [ ] **GitHub Actions:** Go to the "Actions" tab in your GitHub repo and show the successful green checkmark for the "Deploy to Hugging Face Spaces" workflow.
- [ ] **Hugging Face Build Logs:** In your Space, click on the "Logs" tab and show the successful build/container logs.
- [ ] **Hugging Face Files:** Show the `Dockerfile` and `README.md` (with YAML metadata) inside the Hugging Face files tab.

### 3. Monitoring Screenshots
- [ ] **UptimeRobot:** Show the dashboard showing "100% Uptime" for your Resume Analyzer URL.
- [ ] **GitHub Commit History:** Show your recent commits like "chore: update deployment workflow" and "docs: add hugging face space metadata".

### 4. Technical Screenshots (Optional but Recommended)
- [ ] **Local Terminal:** `docker-compose up` running successfully.
- [ ] **SQLite Database:** A screenshot of the `App/resume_data.db` file in your VS Code explorer.
